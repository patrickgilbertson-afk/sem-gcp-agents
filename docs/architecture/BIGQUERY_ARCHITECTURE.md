# BigQuery Table Architecture

## Overview

The `sem_agents` dataset stores all agent operations, recommendations, and configuration. The `sem_ads_raw` dataset (managed by Google Ads Data Transfer Service) contains the source Google Ads data.

## Dataset Structure

```
sem_ads_raw/          # Managed by Google Ads Data Transfer (88 tables)
└── [Google Ads tables with _YYYYMMDD suffix]

sem_agents/           # Managed by Terraform
├── agent_recommendations
├── agent_audit_log
├── agent_state
├── brand_guidelines
├── agent_config
├── recommendation_history
├── llm_calls
├── approval_events
├── performance_metrics
├── campaign_taxonomy (NEW - Phase 2.5)
├── quality_score_history (NEW - Phase 2.5)
└── landing_page_audits (NEW - Phase 2.5)
```

## Core Tables

### 1. agent_recommendations

**Purpose**: Store all recommendations generated by agents before and after approval

**Partitioning**: Daily by `created_at`
**Clustering**: `agent_type`, `status`, `approval_status`
**Retention**: 2 years

```sql
CREATE TABLE sem_agents.agent_recommendations (
  -- Identity
  id STRING NOT NULL,
  run_id STRING NOT NULL,
  agent_type STRING NOT NULL,

  -- Timestamps
  created_at TIMESTAMP NOT NULL,
  approved_at TIMESTAMP,
  applied_at TIMESTAMP,

  -- Content
  title STRING NOT NULL,
  description STRING,
  rationale STRING,
  impact_estimate STRING,
  risk_level STRING NOT NULL DEFAULT 'low',

  -- Action
  action_type STRING NOT NULL,
  action_params JSON,

  -- Status tracking
  status STRING NOT NULL,
  approval_status STRING,
  approved_by STRING,
  approved_by_name STRING,

  -- Execution results
  applied_result JSON,
  error_message STRING,

  -- Metadata
  campaign_id STRING,
  ad_group_id STRING,
  metadata JSON,

  -- Slack tracking
  slack_message_ts STRING,
  slack_channel_id STRING
)
PARTITION BY DATE(created_at)
CLUSTER BY agent_type, status, approval_status;

-- Indexes for common queries
CREATE SEARCH INDEX recommendations_search
ON sem_agents.agent_recommendations(ALL COLUMNS);
```

**Key Queries**:
```sql
-- Pending approvals
SELECT * FROM sem_agents.agent_recommendations
WHERE status = 'awaiting_approval'
  AND DATE(created_at) >= CURRENT_DATE() - 7
ORDER BY created_at DESC;

-- Agent performance
SELECT
  agent_type,
  DATE(created_at) as date,
  COUNT(*) as total_recommendations,
  COUNTIF(approval_status = 'approved') as approved,
  COUNTIF(applied_at IS NOT NULL) as applied,
  COUNTIF(error_message IS NOT NULL) as errors
FROM sem_agents.agent_recommendations
WHERE DATE(created_at) >= CURRENT_DATE() - 30
GROUP BY 1, 2
ORDER BY 1, 2 DESC;
```

### 2. agent_audit_log

**Purpose**: Complete audit trail of every agent action

**Partitioning**: Daily by `timestamp`
**Clustering**: `agent_type`, `event_type`, `run_id`
**Retention**: 1 year

```sql
CREATE TABLE sem_agents.agent_audit_log (
  -- Identity
  run_id STRING NOT NULL,
  agent_type STRING NOT NULL,
  event_type STRING NOT NULL,

  -- Timestamp
  timestamp TIMESTAMP NOT NULL,

  -- Event details
  details JSON,

  -- Context
  user_id STRING,
  session_id STRING,
  parent_run_id STRING,

  -- Performance
  duration_ms INT64,
  tokens_used INT64,
  cost_usd FLOAT64,

  -- Error tracking
  error_code STRING,
  error_message STRING,
  stack_trace STRING
)
PARTITION BY DATE(timestamp)
CLUSTER BY agent_type, event_type, run_id;
```

**Key Queries**:
```sql
-- Recent errors
SELECT
  timestamp,
  agent_type,
  event_type,
  error_message
FROM sem_agents.agent_audit_log
WHERE event_type = 'error'
  AND DATE(timestamp) >= CURRENT_DATE() - 1
ORDER BY timestamp DESC;

-- Agent run timeline
SELECT
  event_type,
  timestamp,
  duration_ms,
  JSON_EXTRACT_SCALAR(details, '$.recommendation_count') as rec_count
FROM sem_agents.agent_audit_log
WHERE run_id = @run_id
ORDER BY timestamp;
```

## Extended Tables (NEW)

### 6. recommendation_history

**Purpose**: Track the full lifecycle of recommendations with state transitions

**Partitioning**: Daily by `event_timestamp`
**Clustering**: `recommendation_id`, `event_type`
**Retention**: 2 years

```sql
CREATE TABLE sem_agents.recommendation_history (
  -- Identity
  history_id STRING NOT NULL,
  recommendation_id STRING NOT NULL,
  run_id STRING NOT NULL,

  -- Event
  event_type STRING NOT NULL,  -- 'created', 'approved', 'rejected', 'applied', 'failed', 'rolled_back'
  event_timestamp TIMESTAMP NOT NULL,

  -- State transition
  from_status STRING,
  to_status STRING,

  -- Actor
  actor_type STRING,  -- 'system', 'user', 'scheduler'
  actor_id STRING,
  actor_name STRING,

  -- Details
  event_details JSON,

  -- Performance impact (if available)
  before_metrics JSON,
  after_metrics JSON,

  -- Rollback info
  is_rollback BOOL DEFAULT FALSE,
  rollback_reason STRING,
  original_history_id STRING
)
PARTITION BY DATE(event_timestamp)
CLUSTER BY recommendation_id, event_type;
```

**Key Queries**:
```sql
-- Recommendation lifecycle
SELECT
  event_timestamp,
  event_type,
  from_status,
  to_status,
  actor_name,
  event_details
FROM sem_agents.recommendation_history
WHERE recommendation_id = @recommendation_id
ORDER BY event_timestamp;

-- Approval velocity
SELECT
  DATE(event_timestamp) as date,
  AVG(TIMESTAMP_DIFF(event_timestamp,
    LAG(event_timestamp) OVER (PARTITION BY recommendation_id ORDER BY event_timestamp),
    MINUTE)) as avg_approval_time_minutes
FROM sem_agents.recommendation_history
WHERE event_type = 'approved'
  AND DATE(event_timestamp) >= CURRENT_DATE() - 30
GROUP BY 1
ORDER BY 1 DESC;
```

### 7. llm_calls

**Purpose**: Track all LLM API calls for cost monitoring and debugging

**Partitioning**: Daily by `timestamp`
**Clustering**: `provider`, `model`, `agent_type`
**Retention**: 90 days

```sql
CREATE TABLE sem_agents.llm_calls (
  -- Identity
  call_id STRING NOT NULL,
  run_id STRING NOT NULL,
  agent_type STRING NOT NULL,

  -- Provider details
  provider STRING NOT NULL,  -- 'anthropic', 'google'
  model STRING NOT NULL,

  -- Timestamp
  timestamp TIMESTAMP NOT NULL,

  -- Request
  prompt_tokens INT64,
  prompt_hash STRING,  -- SHA256 of prompt for deduplication
  system_prompt_hash STRING,
  temperature FLOAT64,
  max_tokens INT64,

  -- Response
  completion_tokens INT64,
  total_tokens INT64,
  response_time_ms INT64,

  -- Cost
  cost_usd FLOAT64,

  -- Quality
  stop_reason STRING,
  finish_reason STRING,

  -- Error tracking
  error_code STRING,
  error_message STRING,
  retry_count INT64 DEFAULT 0,

  -- Portkey metadata
  portkey_request_id STRING,
  portkey_trace_id STRING,
  cache_hit BOOL DEFAULT FALSE,

  -- Context
  session_id STRING,
  user_id STRING
)
PARTITION BY DATE(timestamp)
CLUSTER BY provider, model, agent_type;
```

**Key Queries**:
```sql
-- Daily LLM costs
SELECT
  DATE(timestamp) as date,
  provider,
  model,
  COUNT(*) as call_count,
  SUM(total_tokens) as total_tokens,
  SUM(cost_usd) as total_cost,
  AVG(response_time_ms) as avg_response_time_ms,
  COUNTIF(cache_hit) as cache_hits
FROM sem_agents.llm_calls
WHERE DATE(timestamp) >= CURRENT_DATE() - 30
GROUP BY 1, 2, 3
ORDER BY 1 DESC, 5 DESC;

-- Most expensive prompts
SELECT
  prompt_hash,
  COUNT(*) as call_count,
  AVG(prompt_tokens) as avg_prompt_tokens,
  SUM(cost_usd) as total_cost,
  agent_type,
  model
FROM sem_agents.llm_calls
WHERE DATE(timestamp) >= CURRENT_DATE() - 7
GROUP BY 1, 5, 6
ORDER BY 4 DESC
LIMIT 20;
```

### 8. approval_events

**Purpose**: Detailed tracking of user approval interactions

**Partitioning**: Daily by `event_timestamp`
**Clustering**: `user_id`, `decision`
**Retention**: 1 year

```sql
CREATE TABLE sem_agents.approval_events (
  -- Identity
  event_id STRING NOT NULL,
  approval_request_id STRING NOT NULL,
  recommendation_id STRING NOT NULL,
  run_id STRING NOT NULL,

  -- Timestamp
  event_timestamp TIMESTAMP NOT NULL,
  request_sent_at TIMESTAMP NOT NULL,

  -- User
  user_id STRING NOT NULL,
  user_name STRING,
  user_email STRING,

  -- Decision
  decision STRING NOT NULL,  -- 'approved', 'rejected', 'deferred', 'timeout'
  decision_reason STRING,

  -- Interaction details
  interaction_type STRING,  -- 'button_click', 'modal_submit', 'slash_command'
  time_to_decision_seconds INT64,

  -- Slack details
  slack_channel_id STRING,
  slack_message_ts STRING,
  slack_thread_ts STRING,

  -- Batch info
  is_bulk_approval BOOL DEFAULT FALSE,
  bulk_count INT64,

  -- Edits (for ad copy)
  had_edits BOOL DEFAULT FALSE,
  edit_details JSON,

  -- Context
  device_type STRING,
  timezone STRING
)
PARTITION BY DATE(event_timestamp)
CLUSTER BY user_id, decision;
```

**Key Queries**:
```sql
-- User approval patterns
SELECT
  user_name,
  decision,
  COUNT(*) as decision_count,
  AVG(time_to_decision_seconds) / 60 as avg_decision_time_minutes,
  COUNTIF(is_bulk_approval) as bulk_approvals
FROM sem_agents.approval_events
WHERE DATE(event_timestamp) >= CURRENT_DATE() - 30
GROUP BY 1, 2
ORDER BY 1, 3 DESC;

-- Approval bottlenecks
SELECT
  DATE(event_timestamp) as date,
  agent_type,
  COUNT(*) as total_requests,
  COUNTIF(decision = 'timeout') as timeouts,
  AVG(time_to_decision_seconds) / 3600 as avg_hours_to_decision
FROM sem_agents.approval_events ae
JOIN sem_agents.agent_recommendations ar
  ON ae.recommendation_id = ar.id
WHERE DATE(event_timestamp) >= CURRENT_DATE() - 30
GROUP BY 1, 2
ORDER BY 1 DESC, 4 DESC;
```

### 9. performance_metrics

**Purpose**: Track actual campaign performance impact of applied recommendations

**Partitioning**: Daily by `metric_date`
**Clustering**: `recommendation_id`, `metric_type`
**Retention**: 2 years

```sql
CREATE TABLE sem_agents.performance_metrics (
  -- Identity
  metric_id STRING NOT NULL,
  recommendation_id STRING NOT NULL,
  campaign_id STRING NOT NULL,
  ad_group_id STRING,

  -- Metric details
  metric_date DATE NOT NULL,
  metric_type STRING NOT NULL,  -- 'impressions', 'clicks', 'cost', 'conversions', etc.

  -- Values
  before_value FLOAT64,
  after_value FLOAT64,
  change_value FLOAT64,
  change_percent FLOAT64,

  -- Time periods
  before_period_start DATE,
  before_period_end DATE,
  after_period_start DATE,
  after_period_end DATE,

  -- Statistical significance
  is_statistically_significant BOOL,
  p_value FLOAT64,
  confidence_level FLOAT64,

  -- Metadata
  measured_at TIMESTAMP NOT NULL,
  measurement_method STRING,

  -- Rollup flags
  is_daily BOOL DEFAULT TRUE,
  is_weekly_summary BOOL DEFAULT FALSE,
  is_monthly_summary BOOL DEFAULT FALSE
)
PARTITION BY metric_date
CLUSTER BY recommendation_id, metric_type;
```

**Key Queries**:
```sql
-- Recommendation impact analysis
SELECT
  r.agent_type,
  r.action_type,
  pm.metric_type,
  COUNT(DISTINCT pm.recommendation_id) as recommendation_count,
  AVG(pm.change_percent) as avg_change_percent,
  SUM(pm.change_value) as total_change_value,
  COUNTIF(pm.is_statistically_significant) as significant_count
FROM sem_agents.performance_metrics pm
JOIN sem_agents.agent_recommendations r
  ON pm.recommendation_id = r.id
WHERE pm.metric_date >= CURRENT_DATE() - 30
  AND pm.is_daily = TRUE
GROUP BY 1, 2, 3
ORDER BY 7 DESC;

-- ROI of agent recommendations
WITH cost_impact AS (
  SELECT
    recommendation_id,
    SUM(CASE WHEN metric_type = 'cost' THEN change_value ELSE 0 END) as cost_change,
    SUM(CASE WHEN metric_type = 'conversions' THEN change_value ELSE 0 END) as conversion_change,
    SUM(CASE WHEN metric_type = 'conversion_value' THEN change_value ELSE 0 END) as revenue_change
  FROM sem_agents.performance_metrics
  WHERE metric_date BETWEEN DATE_SUB(CURRENT_DATE(), INTERVAL 30 DAY) AND CURRENT_DATE()
  GROUP BY 1
)
SELECT
  ar.agent_type,
  COUNT(*) as recommendations_with_impact,
  SUM(ci.revenue_change) as total_revenue_impact,
  SUM(ci.cost_change) as total_cost_impact,
  SUM(ci.revenue_change) - SUM(ci.cost_change) as net_profit_impact,
  AVG(ci.conversion_change) as avg_conversion_lift
FROM cost_impact ci
JOIN sem_agents.agent_recommendations ar
  ON ci.recommendation_id = ar.id
WHERE ar.applied_at IS NOT NULL
GROUP BY 1
ORDER BY 5 DESC;
```

## Campaign Taxonomy & Sync Group Tables (NEW - Phase 2.5)

### 10. campaign_taxonomy

**Purpose**: Campaign classification and sync group management for keyword/ad copy propagation

**Partitioning**: None (small dimension table, ~50-200 rows)
**Clustering**: `sync_group`, `campaign_type`
**Retention**: Permanent

```sql
CREATE TABLE sem_agents.campaign_taxonomy (
  -- Identity
  campaign_id STRING NOT NULL,
  campaign_name STRING NOT NULL,
  customer_id STRING NOT NULL,

  -- Classification
  campaign_type STRING NOT NULL,  -- 'nonbrand', 'brand', 'competitor'
  vertical STRING NOT NULL,  -- 'ai_code', 'code', 'languages', 'security', 'brand', 'competitor_github', etc.
  geo STRING NOT NULL,  -- 'US', 'UK', 'DE', 'EMEA', 'Global'

  -- Sync group management
  sync_group STRING NOT NULL,  -- Groups campaigns sharing keywords/ad copy
  management_strategy STRING NOT NULL,  -- 'synced' or 'individual'
  is_template BOOL NOT NULL,  -- Primary campaign in sync group (drives analysis)

  -- Detection metadata
  detection_method STRING NOT NULL,  -- 'auto' or 'manual'
  detection_confidence FLOAT64,  -- 0.0-1.0 for auto-detection quality

  -- Status tracking
  campaign_status STRING,  -- 'ENABLED', 'PAUSED', 'REMOVED' (mirrored from Google Ads)

  -- Audit
  created_at TIMESTAMP NOT NULL,
  updated_at TIMESTAMP NOT NULL,
  updated_by STRING,  -- 'system' or Slack user ID
  notes STRING
)
CLUSTER BY sync_group, campaign_type;

-- Ensure one template per sync group
CREATE UNIQUE INDEX template_per_sync_group
ON sem_agents.campaign_taxonomy(sync_group)
WHERE is_template = TRUE;
```

**Sync Group Derivation Rules**:
- NonBrand → `nonbrand_{vertical}` (e.g., `nonbrand_ai_code`)
- Brand → `brand`
- Competitor → `competitor_{competitor_name}` (e.g., `competitor_github`)

**Management Strategy**:
- NonBrand/Brand → `synced` (changes propagate to all geo campaigns)
- Competitor → `individual` (each evaluated independently)

**Template Selection**: Prefer `US` geo if exists, else first alphabetically

**Key Queries**:
```sql
-- Get all campaigns in a sync group
SELECT
  campaign_id,
  campaign_name,
  geo,
  is_template,
  campaign_status
FROM sem_agents.campaign_taxonomy
WHERE sync_group = @sync_group
  AND campaign_status IN ('ENABLED', 'PAUSED')
ORDER BY is_template DESC, geo;

-- Find unclassified campaigns
SELECT
  c.campaign_id,
  c.campaign_name
FROM `{project}.sem_ads_raw.p_Campaign_{date}` c
LEFT JOIN sem_agents.campaign_taxonomy t
  ON c.campaign_id = t.campaign_id
WHERE t.campaign_id IS NULL;

-- Sync group summary
SELECT
  sync_group,
  campaign_type,
  vertical,
  management_strategy,
  COUNT(*) as campaign_count,
  COUNTIF(is_template) as template_count,
  COUNTIF(campaign_status = 'ENABLED') as enabled_count
FROM sem_agents.campaign_taxonomy
GROUP BY 1, 2, 3, 4
ORDER BY 1;

-- Low confidence auto-detections needing manual review
SELECT
  campaign_id,
  campaign_name,
  campaign_type,
  vertical,
  geo,
  sync_group,
  detection_confidence
FROM sem_agents.campaign_taxonomy
WHERE detection_method = 'auto'
  AND detection_confidence < 0.8
ORDER BY detection_confidence;
```

### 11. quality_score_history

**Purpose**: Daily snapshots of Quality Score and sub-components for trend analysis

**Partitioning**: Daily by `snapshot_date`
**Clustering**: `sync_group`, `campaign_id`
**Retention**: 1 year

```sql
CREATE TABLE sem_agents.quality_score_history (
  -- Identity
  snapshot_id STRING NOT NULL,
  snapshot_date DATE NOT NULL,

  -- Campaign hierarchy
  campaign_id STRING NOT NULL,
  ad_group_id STRING NOT NULL,
  keyword_id STRING NOT NULL,
  keyword_text STRING NOT NULL,
  match_type STRING,

  -- Quality Score components
  quality_score INT64,  -- 1-10 or NULL
  expected_ctr STRING,  -- 'BELOW_AVERAGE', 'AVERAGE', 'ABOVE_AVERAGE'
  ad_relevance STRING,  -- 'BELOW_AVERAGE', 'AVERAGE', 'ABOVE_AVERAGE'
  landing_page_experience STRING,  -- 'BELOW_AVERAGE', 'AVERAGE', 'ABOVE_AVERAGE'

  -- Performance context
  impressions INT64,
  clicks INT64,
  cost_micros INT64,
  conversions FLOAT64,

  -- Taxonomy
  sync_group STRING,
  campaign_type STRING,
  vertical STRING,
  geo STRING,

  -- Snapshot metadata
  created_at TIMESTAMP NOT NULL
)
PARTITION BY snapshot_date
CLUSTER BY sync_group, campaign_id
OPTIONS(
  partition_expiration_days=365
);
```

**Populated by**: Scheduled BigQuery query running daily at 6 AM (before Campaign Health Agent)

```sql
-- Scheduled query to populate quality_score_history
INSERT INTO sem_agents.quality_score_history
SELECT
  GENERATE_UUID() as snapshot_id,
  CURRENT_DATE() as snapshot_date,
  k.campaign_id,
  k.ad_group_id,
  k.criterion_id as keyword_id,
  k.keyword_text,
  k.match_type,
  k.quality_score,
  k.creative_quality_score as expected_ctr,
  k.post_click_quality_score as ad_relevance,
  k.search_predicted_ctr as landing_page_experience,
  k.impressions,
  k.clicks,
  k.cost_micros,
  k.conversions,
  t.sync_group,
  t.campaign_type,
  t.vertical,
  t.geo,
  CURRENT_TIMESTAMP() as created_at
FROM `{project}.sem_ads_raw.p_Keyword_{date}` k
LEFT JOIN sem_agents.campaign_taxonomy t
  ON k.campaign_id = t.campaign_id
WHERE k.status = 'ENABLED';
```

**Key Queries**:
```sql
-- QS drops in last 7 days
SELECT
  keyword_text,
  campaign_id,
  geo,
  sync_group,
  quality_score as current_qs,
  LAG(quality_score, 7) OVER (
    PARTITION BY keyword_id
    ORDER BY snapshot_date
  ) as qs_7d_ago,
  quality_score - LAG(quality_score, 7) OVER (
    PARTITION BY keyword_id
    ORDER BY snapshot_date
  ) as qs_change
FROM sem_agents.quality_score_history
WHERE snapshot_date >= CURRENT_DATE() - 7
  AND quality_score IS NOT NULL
HAVING qs_change < -2
ORDER BY qs_change;

-- QS geo variance for synced campaigns
SELECT
  sync_group,
  keyword_text,
  ARRAY_AGG(
    STRUCT(geo, quality_score, expected_ctr, ad_relevance, landing_page_experience)
    ORDER BY geo
  ) as geo_breakdown,
  MAX(quality_score) - MIN(quality_score) as qs_variance
FROM sem_agents.quality_score_history
WHERE snapshot_date = CURRENT_DATE()
  AND sync_group LIKE 'nonbrand_%'
GROUP BY 1, 2
HAVING qs_variance >= 3
ORDER BY qs_variance DESC;

-- Sub-component degradation analysis
SELECT
  campaign_id,
  keyword_text,
  quality_score,
  expected_ctr,
  ad_relevance,
  landing_page_experience,
  cost_micros / 1000000 as cost_usd
FROM sem_agents.quality_score_history
WHERE snapshot_date = CURRENT_DATE()
  AND quality_score < 5
  AND cost_micros > 50000000  -- $50 spend
ORDER BY quality_score, cost_usd DESC;
```

### 12. landing_page_audits

**Purpose**: Landing page health checks, performance scores, and content relevance analysis

**Partitioning**: Daily by `audit_date`
**Clustering**: `url_hash`, `sync_group`
**Retention**: 1 year

```sql
CREATE TABLE sem_agents.landing_page_audits (
  -- Identity
  audit_id STRING NOT NULL,
  audit_date DATE NOT NULL,

  -- URL tracking
  url STRING NOT NULL,
  url_hash STRING NOT NULL,  -- SHA256(url) for deduplication

  -- PageSpeed Insights metrics
  performance_score FLOAT64,  -- 0-100
  fcp_ms INT64,  -- First Contentful Paint
  lcp_ms INT64,  -- Largest Contentful Paint
  cls FLOAT64,  -- Cumulative Layout Shift
  mobile_friendly BOOL,

  -- Accessibility
  is_accessible BOOL,
  http_status_code INT64,
  redirect_chain ARRAY<STRING>,

  -- Content analysis
  content_hash STRING,  -- SHA256 of page content for change detection
  content_relevance_score FLOAT64,  -- 1-10 from LLM analysis
  keyword_alignment_score FLOAT64,  -- 1-10 keyword coverage score

  -- Improvement recommendations (JSON array)
  improvement_suggestions JSON,

  -- Campaign associations
  sync_groups ARRAY<STRING>,  -- All sync groups using this URL
  campaign_ids ARRAY<STRING>,  -- All campaigns using this URL
  keyword_count INT64,  -- Number of keywords pointing to this URL

  -- Audit metadata
  created_at TIMESTAMP NOT NULL,
  next_audit_date DATE  -- NULL = audit on next scheduled run
)
PARTITION BY audit_date
CLUSTER BY url_hash, sync_group
OPTIONS(
  partition_expiration_days=365
);
```

**Deduplication Strategy**: URLs used across multiple campaigns/geos in a sync group are audited once per week. `url_hash` enables fast lookup for cache checks.

**Key Queries**:
```sql
-- URLs with critical issues
SELECT
  url,
  sync_groups,
  performance_score,
  is_accessible,
  http_status_code,
  content_relevance_score,
  keyword_count,
  improvement_suggestions
FROM sem_agents.landing_page_audits
WHERE audit_date = CURRENT_DATE()
  AND (
    performance_score < 50
    OR is_accessible = FALSE
    OR content_relevance_score < 5
  )
ORDER BY keyword_count DESC, performance_score;

-- URLs needing re-audit (cached for 7 days)
SELECT
  url,
  url_hash,
  MAX(audit_date) as last_audit_date,
  CURRENT_DATE() - MAX(audit_date) as days_since_audit
FROM sem_agents.landing_page_audits
GROUP BY 1, 2
HAVING days_since_audit >= 7;

-- Content relevance by sync group
SELECT
  sg as sync_group,
  COUNT(DISTINCT url) as unique_urls,
  AVG(content_relevance_score) as avg_relevance,
  AVG(performance_score) as avg_performance,
  COUNTIF(performance_score < 50) as slow_urls,
  COUNTIF(content_relevance_score < 5) as low_relevance_urls
FROM sem_agents.landing_page_audits,
  UNNEST(sync_groups) as sg
WHERE audit_date >= CURRENT_DATE() - 7
GROUP BY 1
ORDER BY 5 DESC, 6 DESC;

-- Improvement recommendations summary
SELECT
  url,
  sync_groups,
  JSON_EXTRACT_ARRAY(improvement_suggestions, '$') as suggestions,
  performance_score,
  content_relevance_score
FROM sem_agents.landing_page_audits
WHERE audit_date = CURRENT_DATE()
  AND JSON_EXTRACT_ARRAY(improvement_suggestions, '$') IS NOT NULL
ORDER BY ARRAY_LENGTH(JSON_EXTRACT_ARRAY(improvement_suggestions, '$')) DESC;
```

## Views for Common Queries

### Daily Agent Performance Dashboard
```sql
CREATE OR REPLACE VIEW sem_agents.v_daily_agent_performance AS
SELECT
  DATE(ar.created_at) as date,
  ar.agent_type,
  COUNT(DISTINCT ar.run_id) as runs,
  COUNT(*) as recommendations_generated,
  COUNTIF(ar.approval_status = 'approved') as approved,
  COUNTIF(ar.approval_status = 'rejected') as rejected,
  COUNTIF(ar.applied_at IS NOT NULL) as applied,
  COUNTIF(ar.error_message IS NOT NULL) as errors,
  SUM(lc.cost_usd) as llm_cost_usd,
  AVG(ae.time_to_decision_seconds) / 3600 as avg_approval_hours
FROM sem_agents.agent_recommendations ar
LEFT JOIN sem_agents.llm_calls lc
  ON ar.run_id = lc.run_id
LEFT JOIN sem_agents.approval_events ae
  ON ar.id = ae.recommendation_id
WHERE DATE(ar.created_at) >= CURRENT_DATE() - 90
GROUP BY 1, 2;
```

### Active Recommendations Pending Approval
```sql
CREATE OR REPLACE VIEW sem_agents.v_pending_approvals AS
SELECT
```

**Key Design Principles**:
1. **Partition everything by date** for query performance and cost control
2. **Cluster by access patterns** (agent_type, status, user_id)
3. **JSON for flexible metadata** that doesn't need indexing
4. **Separate history tables** to avoid bloat in primary tables
5. **Retention policies** to auto-delete old data
6. **Materialized views** for expensive aggregations
